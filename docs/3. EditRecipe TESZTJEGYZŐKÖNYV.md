# Receptmegosztó közösségi platform – „Edit Recipe” modul manuális tesztjegyzőkönyv

Tesztet végezte: Benkő Bence  
Dátum: 2025.12.11.  
Talált hibák száma: 2
- Modul megnyitási hiba (adatbázis tömeges betöltés hibás adatokkal) – javítva
- Description mező nem kötelező az Edit modulban – javítva

## 1. Teszt célja és terjedelme

A teszt célja a bejelentkezett felhasználó számára elérhető „Edit Recipe” modul működésének manuális ellenőrzése:
- Modul megnyitása receptkártya „Edit” akciójából
- Kötelező mezők validációja (üres mentés tiltása, hibaüzenetek)
- Description mező kötelezőségének és viselkedésének vizsgálata
- Hozzávalók (materials) és lépések (preparation steps) kezelése (hozzáadás, törlés)
- Változtatások mentése és visszaigazolás

## 2. Tesztkörnyezet és erőforrások

| Elem               | Konfiguráció / Leírás                           |
|--------------------|--------------------------------------------------|
| Operációs rendszer | Windows 10                                       |
| Böngésző           | Google Chrome                                    |
| Tesztelési mód     | Manuális (bejelentkezett felhasználó)           |
| Tesztelési célpont | Receptmegosztó platform (https://localhost:7224/) |
| Kiinduló oldal     | Főoldal (Home), receptkártya „három csík” menü → Edit

Megjegyzés: A teszt során az adatbázis tömeges betöltéséből származó hibás adatok azonosításra kerültek és javítva lettek, ezt követően a modul megnyílt.

## 3. Tesztelési eljárás

Általános lépések:
1. Bejelentkezés sikeresen.
2. Főoldalon egy receptnél a „három csík” (overflow) menü megnyitása, „Edit” választása.
3. Edit modul megnyitásának ellenőrzése (adatok előtöltése – pre-fill).
4. Kötelező mezők üresen hagyása és mentés kísérlete: hibaüzenetek ellenőrzése.
5. Description mező viselkedésének vizsgálata (kötelezőség).
6. Hozzávalók és lépések hozzáadása, törlése.
7. Mentés és eredmény megfigyelése (sikeres mentés, lista/oldal frissítése).

## 4. Tesztesetek és eredmények

### TC-01 – Edit modul megnyitása (kezdeti próbálkozás)
- Akció: „Edit” kattintás receptkártyáról.
- Elvárt: Edit modul megnyílik, adatok előtöltve.
- Tényleges: Modul nem nyílt meg; hiba oka később azonosítva (hibás, tömegesen betöltött adatbázis rekordok).
- Állapot: Nem felelt meg

### Hiba-01 – Hibás adatbázis adatok miatt modul nem nyílt meg
- Leírás: Tömeges betöltés során hibás adatok kerültek az adatbázisba, ez megakadályozta az Edit modul megnyitását.
- Hatás: Edit funkció használhatatlan, szerkesztés nem indítható.
- Megoldás: Hibás rekordok javítása/adattisztítás. Javítás után a modul megnyílt.

### TC-02 – Edit modul megnyitása (javítás után)
- Akció: „Edit” kattintás javítás után.
- Elvárt: Edit modul megnyílik, adatok előtöltve.
- Tényleges: Modul rendben megnyílt, pre-fill helyes.
- Állapot: Megfelelt

### TC-03 – Kötelező mezők üresen hagyása (validáció)
- Bemenet: Kötelező mezők üresen hagyása (Dish Name, Category, Cook Time, Preparation Time, Portions).
- Elvárt: Hibaüzenet(ek) jelennek meg; mentés blokkolva.
- Tényleges: Minden kötelező mezőnél megfelelő hibaüzenet; mentés nem engedélyezett üresen.
- Állapot: Megfelelt

### TC-04 – Description mező viselkedése az Edit modulban
- Bemenet: Description üresen hagyva.
- Elvárt: Következetesen kötelező (az Add modulban már kötelezővé téve), ezért itt is kötelező legyen; üresen hibaüzenet.
- Tényleges: Description nem kötelező; üresen mentés engedélyezett. Ez inkonzisztens a rendszerrel.
- Állapot: Nem felelt meg

### Hiba-02 – Description mező nem kötelező az Edit modulban
- Leírás: Az Edit modulban a Description mező nem kötelező, ellentétben az Add modul állapotával és az elvárt konzisztenciával.
- Hatás: Inkonzisztens adatkezelés; leírás nélküli receptek kerülhetnek a rendszerbe.
- Javaslat: Tegyük kötelezővé a Description mezőt az Edit modulban is; egységes front-end és backend validációval.

### TC-05 – Hozzávalók (materials) törlése és hozzáadása
- Akció: Meglévő hozzávaló törlése; új hozzávaló hozzáadása (dropdown + „Hozzáadás”).
- Elvárt: Törölt elem eltűnik; új elem megjelenik; hiba nélkül menthető.
- Tényleges: Törlés és hozzáadás működött; lista megfelelően frissült.
- Állapot: Megfelelt

### TC-06 – Preparation steps törlése és hozzáadása
- Akció: Meglévő lépések törlése; új lépések hozzáadása.
- Elvárt: Törlés és hozzáadás működik; lépések helyesen jelennek meg és menthetők.
- Tényleges: Műveletek rendben működtek; mentés után változások megmaradtak.
- Állapot: Megfelelt

### TC-07 – Mentés funkció
- Akció: Változtatások mentése.
- Elvárt: Sikeres mentés; frissítések láthatók a recept részleteinél/listában; hiba nélkül záródik a modul.
- Tényleges: Mentés sikeres; változtatások érvényesültek.
- Állapot: Megfelelt

## 5. Következtetések és ajánlások

- A kezdeti modul-megnyitási hiba adatminőségi probléma volt; javítás után az Edit modul működik.
- Kötelező mezők validációja megfelelő, ugyanakkor a Description mező kötelezőségét az Edit modulban egységesíteni kellett.
- Hozzávalók és lépések kezelése (törlés/hozzáadás) stabil; mentés funkció megbízható.
